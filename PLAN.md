# Plan: TIL CMS + Docusaurus site for Dave Leeds' Kotlin Coroutines course

## Context

Learning notes ("Today I Learned") for Dave Leeds' *Kotlin Coroutines and Concurrency*
course, to be shared with the team. Two parts:

1. A **local CMS** - a small web app with a menu of every course lesson. Clicking a lesson
   opens a page with structured text fields. Each field has a **"Rewrite with Claude"** button
   that fixes only grammar/spelling/punctuation without changing meaning.
2. A **Docusaurus site** that publishes those notes. The CMS writes Markdown files directly
   into the Docusaurus `docs/` folder, so saving a note = updating the docs.

Decisions:
- **Claude rewrite** = shell out to the `claude` CLI (`claude -p`) - uses the existing
  subscription, no API key, no per-call cost.
- **Note fields** = Key Takeaways · What I Still Don't Understand · General Notes ·
  Code Snippets & Gotchas.
- **Layout** = separate subfolders (`docs-site/`, `cms/`, `kotlin/`) under a root that
  orchestrates everything. Runs locally on both macOS and Windows. No hosting yet.

## Repo layout

```
Dave-Leeds-Coroutines-Course/
├── package.json                 # root: npm workspaces + orchestration scripts
├── README.md                    # how to run everything
├── PLAN.md                      # this file
├── course-structure.json        # single source of truth: 12 modules → lessons (title+slug)
├── docs-site/                   # Docusaurus (classic template)
│   └── docs/                    # generated lesson .md files land here (per-module folders)
├── cms/
│   ├── server.js                # Express: serves UI, read/save notes, /api/rewrite
│   ├── lib/markdown.js          # generate MD from fields + parse MD back into fields
│   ├── lib/claude.js            # spawn `claude -p`, cross-platform
│   └── public/                  # no-build SPA: index.html, app.js, styles.css
└── kotlin/                      # practice Gradle projects (README only for now)
```

## Components

1. **`course-structure.json`** - all 12 modules / ~55 lessons in course order. Drives both the
   CMS menu and the Docusaurus folder structure.
2. **Docusaurus** - classic template, autogenerated sidebar, `routeBasePath: '/'`, docs as the
   site root; each module folder gets a `_category_.json` (label + position). Kotlin syntax
   highlighting enabled.
3. **CMS** - Express backend (`/api/structure`, `/api/note/...` GET+POST, `/api/rewrite`) +
   a no-build vanilla SPA. Markdown is the single source of truth (frontmatter + fixed `##`
   sections; parsed back into fields on load). Rewrite spawns the `claude` CLI via stdin,
   cross-platform (`shell:true`).
4. **Kotlin** - folder + README; no build scaffolded.
5. **Root** - npm workspaces (`docs-site`, `cms`); `npm run dev` runs both via `concurrently`.

## Verification

1. `npm install` at root installs both workspaces.
2. `npm run dev` → CMS at :4000, Docusaurus at :3000.
3. In the CMS: pick a lesson, type text, **Rewrite with Claude** fixes grammar (meaning intact).
4. **Save** writes `docs-site/docs/<module>/<lesson>.md` with frontmatter + sections.
5. Docusaurus shows the lesson in the sidebar under its module.
6. Re-open the lesson in the CMS → fields repopulate from the saved MD (round-trip).
